# ONEPIC

AI 기반 상품 인식 스마트 쇼핑 서비스

> 이미지 분석 결과를 바탕으로 상품을 탐색하고, 상세 조회부터 결제 흐름까지 연결하는 프로젝트입니다.

## 1) 프로젝트 한눈에 보기

- **목표:** AI 분석 결과를 실제 쇼핑 경험으로 연결
- **핵심 기능:** 상품 조회, 상세 조회, 결제 흐름, 외부 AI 서버 연동
- **기간/형태:** 팀 프로젝트
- **내 역할:** **DB 설계 및 백엔드 데이터 흐름 구현 중심**

---

## 2) 서비스 흐름

1. 사용자가 이미지를 업로드
2. AI 서버가 이미지 분석 후 키워드/카테고리 반환
3. 백엔드가 분석 결과 기반으로 상품 후보 조회
4. 사용자가 상품 상세 확인
5. 장바구니/주문/결제 순서로 진행
6. 주문 데이터 저장 및 후속 조회 API 제공

---

## 3) 내가 맡은 부분 (Database)

### 3-1. DB 모델링

- 상품, 카테고리, 주문, 주문상세, 사용자 중심으로 스키마 설계
- 조회/주문 시나리오 기준으로 테이블 관계 정의
- 중복 최소화와 조회 성능 균형을 고려한 구조 설계

### 3-2. 데이터 흐름 설계

- AI 분석 결과 → 상품 조회 조건으로 변환
- 상품 상세/재고/가격 정보 조회 API와 DB 연결
- 주문 생성 시 주문/주문상세 데이터 일관 저장

### 3-3. 트랜잭션 & 무결성

- 주문 처리 구간 트랜잭션 적용
- 예외 발생 시 롤백 처리로 데이터 정합성 유지
- FK/제약조건 기반 무결성 관리

### 3-4. 성능 관점 개선

- 자주 조회되는 조건(카테고리, 상품 식별자 등) 인덱스 최적화
- API 응답에 맞춘 조회 쿼리 단순화
- 불필요한 중복 조회 최소화

---

## 4) 기술 스택

- **Backend:** Java, Spring Boot, JPA
- **Database:** MySQL
- **Infra/DevOps:** AWS, Docker
- **Collaboration:** Git, GitHub

---

## 5) 회고

- 기능 구현뿐 아니라 **데이터 구조가 서비스 품질에 직접 영향**을 준다는 점을 체감
- 초기 모델링 단계에서 조회 패턴/트랜잭션 단위를 먼저 정의하는 습관을 학습
- 앞으로는 대용량 데이터 기준의 인덱스/쿼리 튜닝을 더 고도화할 계획
